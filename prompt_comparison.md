# プロンプト比較分析

## 絶対厳守の3要素

1. **スタイル**（文体・表現方法）
2. **文字数**（3,500〜5,000字）
3. **質**（顧客固有の内容、深い共感）

---

## 詳細比較

### 1. スタイル（文体・表現方法）

#### 現在のプロンプト（divination-chapter-based.ts）
```
- 非常に感情的で深く共感する語りかけ
- スピリチュアルな表現を多用（「視えました」「波動が伝わってきました」「魂が叫んでいます」）
- 顧客の内面を深く推測して具体的に描写
```

**問題点:**
- 具体的な文体ルールが不足
- 短文の使い方、心の声の表現方法が不明確
- リズムの作り方が指定されていない

#### 新しいプロンプト
```
1. 短文の連続
   ✅ 「でも、次の瞬間。不安が襲ってくる。」
   ✅ 「そして、片付けを終えて、スマホを開く。」
   ❌ 「でも次の瞬間不安が襲ってきて、片付けを終えてスマホを開きました。」

2. 心の声の終わり方
   ✅ 「今日もよく頑張ったな」って。
   ❌ 「今日もよく頑張ったな、と思う。」
   必ず「〜って。」で終わる
   引用符「」内に読点（、）は使わない

3. 畳みかけるリズム
   ✅ 「朝起きた時も、同じことを考えている。」
   ✅ 「施術の合間も、同じことを考えている。」
   ✅ 「夜寝る前も、同じことを考えている。」
   同じ構文を3回繰り返す

4. 段落の区切り方
   - 1〜2文ごとに改行
   - 空白行を入れて、呼吸を作る

5. 接続詞の使い方
   - 「でもね」「だから」「そして」を文頭に置く

6. 「視えました」の使い方
   - 必ず独立した段落にする
   - 「視えました。」だけで1行
   - 全体で5回のみ使用

7. 労いの言葉
   - 「本当に、本当によく頑張ってきましたね。」
   - 同じ言葉を2回繰り返す
```

**優位性:**
- ✅ 文体ルールが極めて具体的
- ✅ 実例とNG例を明示
- ✅ リズムの作り方が明確
- ✅ 「視えました」の使用回数を制限（5回のみ）

---

### 2. 文字数制御

#### 現在のプロンプト（章別生成方式）
```
- 導入部: max_tokens 450 → 約300字
- 第一章: max_tokens 900 → 約600字
- 第二章: max_tokens 750 → 約500字
- 第三章: max_tokens 1050 → 約700字
- 第四章: max_tokens 1200 → 約800字
- 第五章: max_tokens 900 → 約600字
- 結び: max_tokens 450 → 約300字
合計: 5,700トークン ≈ 3,990字
```

**問題点:**
- 実際のテスト結果: 2,957字（目標に届かない）
- max_tokensの設定が不正確（日本語は1トークン ≈ 0.7字）

#### 新しいプロンプト
```
1. 導入（300字）
2. 「視えました」パート（600字）
3. 第一章：魂の傷（600字）
4. 第二章：答え合わせ（500字）
5. 第三章：前世（600字）
6. 第四章：使命（700字）
7. 第五章：三つの道（600字）
8. 結び（300字）

合計目標：4,200字
※最終的に3,500〜5,000字の範囲内に収める

【絶対厳守】
- 合計文字数：3,500〜5,000字
- 3,500字未満は絶対にダメ
- 5,000字を絶対に超えてはいけない
```

**優位性:**
- ✅ 文字数の下限と上限を明確に指定
- ✅ 8章構成で各章の役割が明確
- ✅ 「3,500字未満は絶対にダメ」と強調

---

### 3. 質（顧客固有の内容、深い共感）

#### 現在のプロンプト
```
<学習データ>
以下は実際の深層鑑定文の例です。この文体・トーン・雰囲気を完全に再現してください:

${trainingExample}
</学習データ>
```

**問題点:**
- ❌ 学習データの「内容」も真似してしまう可能性が高い
- ❌ 「内容は真似しないで」という指示が不足
- ❌ 顧客固有の内容を作成する指示が弱い

#### 新しいプロンプト
```
【重要】
- 以下の参考例は「構成」と「文体スタイル」を学ぶためのものです
- 参考例の「内容」（早苗さんの悩み、輝雄さんの状況など）は絶対に真似しないでください
- 提供されたチャット履歴から、その顧客固有の内容を作成してください

【絶対に守ること】
- 参考例の「内容」は真似しない
  - ❌ 早苗さんのサロンの話を使わない
  - ❌ 輝雄さんのリタイアの話を使わない
  - ✅ チャット履歴から、その顧客固有の内容を作る

- 参考例の「構成」と「スタイル」は真似する
  - ✅ 短文の連続
  - ✅ 「〜って。」で終わる心の声
  - ✅ 畳みかけるリズム
  - ✅ 段落の区切り方
```

**優位性:**
- ✅ 「内容」と「構成・スタイル」を明確に分離
- ✅ 具体的な禁止事項を明記
- ✅ 顧客固有の内容を作成する指示が強い

## 構成の違い

### 現在のプロンプト（7章構成）
1. 導入部
2. 第一章 - あなたの魂が背負う三つの傷
3. 第二章 - あなたの直感は正しい
4. 第三章 - 前世から続く魂の物語
5. 第四章 - 未来のビジョン
6. 第五章 - 三つの道
7. 結び

### 新しいプロンプト（8章構成）
1. 導入（300字）
2. **「視えました」パート（600字）** ← 新規追加
3. 第一章：魂の傷（600字）
4. 第二章：答え合わせ（500字）
5. 第三章：前世（600字）
6. 第四章：使命（700字）
7. 第五章：三つの道（600字）
8. 結び（300字）

**重要な違い:**
- 新しいプロンプトには「視えました」パートが独立した章として存在
- これは顧客の具体的な状況を描写する重要なパート
- 学習データの文体を最もよく表現している部分

## 必須ワードの比較

### 現在のプロンプト
- 特に指定なし

### 新しいプロンプト
```
【必須ワード】
- 「視えました」：全体で5回のみ
- 「あなたのせいじゃない」：第一章で1回
- 「違うんです」：第二章で1回
- 「無理ないんですよ」：第三章で1回
- 「何も悪くない」：結びで1回
```

**優位性:**
- ✅ 文体の一貫性を保つ
- ✅ スピリチュアルな雰囲気を維持
- ✅ 使用回数を制限して過度な使用を防ぐ

## スピリチュアル用語の比較

### 現在のプロンプト
- 特に制限なし

### 新しいプロンプト
```
【スピリチュアル用語】
- 使用OK：魂、波動、オーラ、エネルギー
- 使用NG：守護霊、霊、宇宙、周波数、チャクラ
```

**優位性:**
- ✅ 学習データの文体に合わせた用語選択
- ✅ 過度にスピリチュアルになりすぎない

## 推奨アクション

**新しいプロンプトを章別生成方式に適用することを強く推奨します。**

理由:
1. **スタイルの精度が圧倒的に高い**（具体的なルール7項目）
2. **文字数制御が明確**（3,500〜5,000字の厳守）
3. **質の向上**（内容と構成の明確な分離）
4. **章別生成方式のメリット**（max_tokensで物理的に制限）

実装方針:
- 8章構成に変更
- 各章で新しいプロンプトのスタイルルールを適用
- max_tokensを調整して3,500〜5,000字を確保
