# 深層鑑定文生成ツール - 公開環境設定ガイド

## 📋 概要

このガイドでは、開発環境で正常に動作している鑑定文生成ツールを、公開環境でも動作させるための手順を説明します。

---

## 🔍 現在の状況

### 開発環境（プレビュー）
✅ **正常に動作**
- 文字数カウント：動作
- 鑑定文生成：成功（4000字前後）
- 環境変数：正しく読み込まれている

### 公開環境
❌ **500エラー発生**
- 文字数カウント：動作
- 鑑定文生成：失敗（500エラー）
- 環境変数：未確認（おそらく未設定）

---

## 🎯 問題の原因

**公開環境で `OPENROUTER_API_KEY` 環境変数が設定されていない**

開発環境では、ローカルの環境変数が使用されますが、公開環境では、Manusプラットフォームが管理する環境変数のみが使用されます。そのため、Management UIのSecretsで明示的に設定する必要があります。

---

## ✅ 解決手順

### ステップ1: 最新のコードを公開

1. **Management UIを開く**
   - ブラウザでプロジェクトのManagement UIにアクセス

2. **Publishボタンをクリック**
   - 画面右上の「Publish」ボタンをクリック

3. **最新のチェックポイントを選択**
   - チェックポイント `3bfb2833` を選択
   - 公開を実行

4. **公開完了を待つ**
   - 数分かかる場合があります
   - 完了するまで待機

---

### ステップ2: 環境変数の確認

1. **デバッグページにアクセス**
   - ブラウザで以下のURLを開く：
   ```
   https://divinationapp-npixyxzz.manus.space/debug-env
   ```

2. **環境変数の状態を確認**
   - 「OpenRouter APIキー設定」の項目を確認
   - ✅ **設定済み**：ステップ3をスキップ
   - ❌ **未設定**：ステップ3へ進む

---

### ステップ3: 環境変数の設定（未設定の場合のみ）

1. **Management UIのSecretsページを開く**
   - Management UI → **Settings** → **Secrets**

2. **新しいSecretを追加**
   - 「Add Secret」または「＋」ボタンをクリック

3. **環境変数を入力**
   ```
   Key: OPENROUTER_API_KEY
   Value: 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
   ```

4. **保存**
   - 「Save」または「保存」ボタンをクリック

5. **サーバーの再起動を待つ**
   - 環境変数を追加すると、サーバーが自動的に再起動します
   - 数分待機

---

### ステップ4: 動作確認

1. **ブラウザのキャッシュをクリア**
   - `Ctrl + Shift + R`（Windows/Linux）
   - `Cmd + Shift + R`（Mac）

2. **デバッグページで再確認**
   ```
   https://divinationapp-npixyxzz.manus.space/debug-env
   ```
   - 「OpenRouter APIキー設定: ✅ 設定済み」と表示されることを確認

3. **鑑定文生成ページにアクセス**
   ```
   https://divinationapp-npixyxzz.manus.space/generate
   ```

4. **テストデータを入力**
   ```
   ①田中花子 女
   [2024/01/15 14:23] 田中花子: こんにちは、鑑定お願いします
   [2024/01/15 14:25] あなた: はじめまして、どのようなことでお悩みですか？
   [2024/01/15 14:27] 田中花子: 最近、仕事でうまくいかないことが多くて...
   ```

5. **「鑑定文を生成」ボタンをクリック**

6. **生成結果を確認**
   - 右側のパネルに鑑定文が表示されることを確認
   - 文字数が3,500〜5,000字であることを確認

---

## 🚨 トラブルシューティング

### 問題1: デバッグページが404エラー

**原因：** 最新のコードが公開されていない

**解決策：**
1. Management UIで「Publish」ボタンをクリック
2. 最新のチェックポイント（3bfb2833）を選択して公開
3. 公開完了後、ブラウザで `Ctrl + Shift + R` を押してキャッシュをクリア

---

### 問題2: デバッグページで「❌ 未設定」と表示

**原因：** 環境変数が設定されていない

**解決策：**
1. Management UI → Settings → Secrets
2. `OPENROUTER_API_KEY` を追加（ステップ3参照）
3. サーバーの再起動を待つ（数分）
4. 再度デバッグページにアクセス

---

### 問題3: 鑑定文生成で500エラー

**原因：** 環境変数が正しく読み込まれていない、またはAPIキーが無効

**解決策：**
1. デバッグページで環境変数を確認
2. APIキーが正しいか確認（73文字、sk-or-v1-で始まる）
3. Management UI → Settings → Secrets で値を再入力
4. サーバーの再起動を待つ

---

### 問題4: ブラウザのコンソールにエラーが表示

**確認方法：**
1. ブラウザで `F12` キーを押して開発者ツールを開く
2. 「Console」タブを選択
3. エラーメッセージを確認

**よくあるエラー：**
- `Failed to load resource: 500`：環境変数が未設定
- `OPENROUTER_API_KEY is not set`：環境変数が未設定
- `Network error`：インターネット接続の問題

## 🔐 セキュリティに関する注意事項

1. **APIキーの管理**
   - APIキーは秘密情報です。他人と共有しないでください
   - Management UIのSecretsに保存されたAPIキーは暗号化されています

2. **環境変数の確認**
   - デバッグページ（/debug-env）は開発用です
   - 本番環境では、このページを削除または保護することを推奨します

3. **アクセス制限**
   - 現在、ログインしたユーザーのみが鑑定文を生成できます
   - 必要に応じて、管理者のみに制限することも可能です
